Github Notları

Temel Kelimeler

  1. repository: kısa ismiyle repo kodlarımızın saklandığı yerdir
  2. naster: depodaki ana proje sürümüdür
  3. branch:git ağaç ile çalışır ve projedeki geliştirmelerimii ağaç dalı mantığı ile ayrı dallarda saklayabiir,sonrasnda master branch de birleştirebilriz
  4. marge:branch lerde yapılan işlemlerin master da birleştirilmesidir.
  5. commit:commit işlemi lokaldeki dosyalarımızı depoya gönderilmek üzere hazırlar
  6. push:dosyalarımızı depoya göndermemii sağlar
  7. pull:depodaki dosyalarımızı çekmemizi sağlar
  8. .gitignore:git in takip etmesini istemediğimiz dosya ve klasörleri belirttiğimiz dosyadır

Vs code da projede bir değişiklik yapıldıktan sonra bunu girhub a göndermek için source control tabını açtı Message alanına yapılan değişiklikle alakalı bir mesaj yazdık Son olarak commit butonuna basıp depoya gönderlmek üzere hazırladık ve sync butonuna basarak push işlemini yaparak değilikliği github a yolladık

Temel Git Komutları

  1. git add veya git add*,git add index.html:Klasöre eklenen dosyaları gönderilmek üzere sahneye alma(stage)
  2. git statu:klasördeki tüm değişiklikleri göster
  3. git commit-m "ilk commit" :depoya gönderilmek için ommit işlemi yapılır
  4. git commit -a -m "mesajımız":staging e ekle ve comitle
